# Design-Partner Outreach

## Who to invite first

- developers who copy code, JSON, URLs, and shell commands every day
- maintainers already using Maccy, Raycast, Alfred, or Paste
- designers or marketers who reuse assets, links, and short text blocks

## Short DM / email script

> I’m working on OpenPaste, a native macOS clipboard manager for developers.
> It’s open source, local-first, and focused on fast recall instead of hype.
> Would you be open to trying a beta build and telling me where it helps or breaks in your real workflow?

## Follow-up if they say yes

> Perfect — I mainly want to learn three things:
> 1. what you copy most often,
> 2. where current clipboard tools annoy you,
> 3. what would make you keep OpenPaste installed after week one.

## 15-minute call plan

1. **2 min** — what they copy all day
2. **5 min** — cold install + first use
3. **5 min** — repeat one real workflow
4. **3 min** — what would keep them using it next week

## What to listen for

- language they naturally use to describe the product
- trust concerns
- install friction
- whether they remember OpenPaste as history, memory, or workflow tool
