# OpenPaste Positioning (6-week freeze)

## Core statement

OpenPaste is a **native, local-first clipboard manager for developers on macOS**.

## Short versions

- **12 words:** Native clipboard manager for developers who want fast recall and local control.
- **30 words:** OpenPaste helps developers recall copied code, JSON, URLs, images, and files quickly with a native macOS UI, keyboard-first flow, local-first defaults, and no telemetry.

## Who this is for

- Developers and open-source maintainers
- DevOps / SRE users handling secrets, commands, and structured text
- Designers or marketers who regularly reuse assets, links, and copy blocks

## Why switch

- **Native macOS feel** instead of a generic cross-platform shell
- **Open source** instead of a closed subscription utility
- **Local-first defaults** with no telemetry
- **Useful for developer workflows today**: fast search, OCR, collections, paste stack, quick edit

## What we say publicly today

- Fast local clipboard recall for code, JSON, URLs, files, images, and prompts
- Keyboard-first navigation with a native SwiftUI/AppKit UI
- Local-first default behavior with privacy guards built in
- Homebrew install + notarized DMG release path

## What we label as “still maturing”

- iCloud sync foundation (premium-gated beta in current builds)
- SQLCipher hardening across build variants
- Auto-update / distribution polish

## What we label as roadmap, not current feature

- Semantic search
- AI actions
- Snippets/templates and text expansion
- Content transformations
- Plugin SDK / marketplace

## Messaging guardrails

### Do say

- native
- local-first
- open source
- keyboard-first
- no telemetry
- built for developers

### Don’t say

- AI-native today
- semantic search today
- plugin-ready today
- encryption-at-rest guaranteed in every build today
